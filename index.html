Proposal 1: One-shot Scenes Interaction

TL; DR One-shot in this context means to estimate the depth information from a single image, and further use this 3D scene information to achieve interactions. Furthermore, combining this with the language-guided human motion generation methods can further extend scene affordance.

Human interaction with one-shot scenes provided by monocular depth estimation. We employ a monocular depth estimation method to generate the RGB point cloud, which is particularly suitable for scenarios with no multi-view or depth camera for 3D reconstruction. Results below are based on an image found from the Internet, and there is only one frame for inference. We generate depth image using the LCM version of Marigold and convert it to RGB point cloud using Open3D. Code is availeble at code. We use MDM to generate language-guided human actions, but in the following examples we use Mixamo's publicly available actions because it is more stable for demo-only use. The human motion scene in the 3D point cloud shown in the video is rendered in Blender.
Note that all the examples on this page are for demonstration purposes only.

A person walks from right to left behind the campus signage.

Is the generated 3D point cloud accurate?
The 3D point cloud is generated from the depth map obtained from the one-shot image by monocular depth estimation, which means it can only represent 3D scene information under the current viewing angle if no other help is available. Nonetheless, this still allows us to create more complex scenes based on simple 2D images.

What are the consequences of the absence of multi-view information?
The treatment of occluded regions remains unexplored. A simple example is the difficulty of telling where the horizon is behind the campus signage in the demo above, which can be a problem for interaction. However, in this scenario language instructions are also given via the 2D image, which is more friendly for specific interactions in the occluded area.

Proposal 2: How depth estimation helps 3D reconstruction

TL; DR We propose to enhance the noisy sparse depth input from estimated depth to support the NeRF optimization.

Roessle, B. et al. (CVPR2022) leverage dense depth priors to constrain NeRF optimization. Firstly, they utilize sparse depth data freely available from the Structure from Motion (SfM) preprocessing step to estimate the camera pose. Then they use depth completion to convert these sparse points into dense depth maps and uncertainty estimates, which are used to guide NeRF optimisation [1]. A successful reconstruction from RGB images requires a large number of input views taken under static conditions, method with dense depth prior enables data-efficient novel view synthesis with less than 20 images, as opposed to 100+ images required before.

Simply using the depth map to regularize all parameters would cause over-smooth and blurry details in 3D Gaussian Radiance Fields. Jiahe Li et al. (CVPR 2024) propose DNGaussian, they propose the Hard and Soft Depth Regularization that achieves a balance between accurate geometry and delicate appearance. And the Global-Local Depth Normalization proposed by them refocuses on local depth differences, eliminates small depth errors and thus helping build a clean surface [2].
